## Styling Details

### Rate-Limit Warning Box
- **Background**: `bg-amber-500/5` (very light amber/yellow tint)
- **Border**: `border border-amber-500/20` (subtle amber border)
- **Text Color**: `text-amber-600` (amber text)
- **Padding**: `px-2 py-1` (compact padding)
- **Border Radius**: `rounded` (slightly rounded corners)
- **Font Size**: `text-xs` (small text, same as other metadata)

### Icon
- **Component**: `<AlertTriangle size={12} />`
- **Color**: Inherits `text-amber-600`
- **Flex**: `flex-shrink-0` (doesn't shrink on small screens)

### Layout
- **Display**: Flexbox with `gap-3` between items
- **Wrapping**: `flex-wrap` enabled for responsive layout
- **Alignment**: `items-center` for vertical centering

## Color Reference

### Health Status Colors (Existing)
- **Healthy**: Green (`text-green-600`, `bg-green-500/10`, `border-green-500/30`)
- **Warning**: Amber (`text-amber-600`, `bg-amber-500/10`, `border-amber-500/30`)
- **Error**: Red (`text-red-600`, `bg-red-500/10`, `border-red-500/30`)

### Rate-Limit Warning Colors (New)
- **Text**: `text-amber-600` (amber/yellow text)
- **Background**: `bg-amber-500/5` (5% opacity amber)
- **Border**: `border-amber-500/20` (20% opacity amber)
- **Icon**: Same as text color

These colors were chosen to:
1. Match the existing "Warning" health status styling
2. Stand out enough to catch attention but not alarm
3. Indicate a temporary/recoverable condition (not critical)
4. Maintain design consistency with the rest of the Admin UI
